Subnet Masks

 

Hosts and Networks

In computer networking, the term 'host' refers to any computer attached to a computer network. Thus a server can be a host and a user's computer can be a host; all computers on the network whether they are a server or client computer, are hosts.

Often large organizations will have more than one network. In some cases they may have several or even more networks. These networks can be connected to each other and often are. Thus there is a need to keep track of which network is which and which host is on which network. This complex task is partly done by using subnet masks. A subnet mask (in it's simplest role) is used to break an IP Address into two parts: the network part and the host part. The network part is often referred to as the 'Network ID' and looks like an ordinary IP Address but often ends with a '0' (for example, 192.158.25.0).

A subnet mask is used to determine the network and host part of an IP Address.

Host Part

To make sense of how this works, first you must look at the subnet mask in its binary format. For example, the subnet mask '255.255.255.0' is shown below:

255 255 255 0
1111 1111 1111 1111 1111 1111 0000 0000

The network part is simply where the ones are and the host part is where the zeros are. For example:

255 255 255 0
1111 1111 1111 1111 1111 1111 0000 0000
Network Part Host Part

Often when discussing subnet masking, an 'n' is used do denote the bit-places for the network part and an 'h' is used to denote the bit-places of the host part. For example, the above subnet mask would be denoted as: nnnnnnnn.nnnnnnnn.nnnnnnnn.hhhhhhhh

(Please note that the color coding I am using here is NOT a standard.)

Here is a table to further demonstrate this point:

255 255 255 0
1111 1111 1111 1111 1111 1111 0000 0000
nnnn nnnn nnnn nnnn nnnn nnnn hhhh hhhh
Network Part Host Part

A short hand notation that is sometimes used for subnet masks like this is: N.N.N.H

 

Default Subnet Masks

IP Addresses are broken up into different 'classes' (click here to see a chart of these classes). The first three classes (A, B, and C) have default subnet masks which are listed below:

Class Network-Host Parts

Default Subnet Mask
A N.H.H.H    255.0.0.0
B N.N.H.H    255.255.0.0
C N.N.N.H    255.255.255.0

 

Sub-Networks and Subnet-Bits

Organizations often need to further subdivide a given network. If an organization receives a complete set of Class B IP Address, for example it receives use of all the IP Address available to Network ID 130.10.0.0 (N.N.H.H), then 65,534 host can be put on that network. Often it is not practical to put that many hosts on a single network so they will create 'sub-networks'. In other words, they will break their network up into parts. These pieces of the original network are called 'sub-networks'.

Here is a chart that shows a Class B subnet mask with a single 'subnet-bit' added:

255 255 128 0
1111 1111 1111 1111 1000 000 0000 0000
nnnn nnnn nnnn nnnn shhh hhhh hhhh hhhh
Network Part Sub/Host Host Part

Notice the single bit that is denoted by a single 's'. This 'subnet-bit' divides the network into two sub-networks as shown in the chart below.

If another 'subnet-bit' were added (nnnnnnnn.nnnnnnnn.sshhhhhh.hhhhhhhh), then the network would be divided into 4 sub-networks. If 3 'subnet-bits' were added (nnnnnnnn.nnnnnnnn.ssshhhhh.hhhhhhhh), then the network would be divided into 8 sub-networks, and so on. The formula for determining the number of sub-networks is:

       (number of subnet-bits)
2
